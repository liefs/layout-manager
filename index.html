liefs-layout-manager CONCEPT PAGE

liefs-layout-manager is (Typescript) javascript code that manages the positions of the elements on the page. This is a basic "concept" outline.

Item refers to liefs-layout-manager.Item
Dragbar refers to liefs-layout-manager.Dragbar
Container refers to liefs-layout-manager.Container
Layout refers to liefs-layout-manager.Layout
Handler refers to liefs-layout-manager.Handler

liefs-layout-manager does the following:

  • An Item (Object/Class) points to an HTML Element. Each Item points to only one Element. It is common for different Items to point to the same Element. This "assignment" Indicates that the position of this HTML element will be governed by liefs-layout-manager (by css Element.style - position, visibility, left, top, width, height). This element, and all its children will ignore what the dom proposes as its postion, and will be "overwritten" to whatever liefs-layout-manager deceides is best. (Detailed later on)
    Item (Object/Class) Handles the creation, and storage of all Item related data.

  • An Item can be extended to be the "root" of a Container. (Details explained later on)
    An Item can be extedend to include other items, as Pages. (Details explained later on)

  • A Container (Object/Class) contains an Array of [row (horizontal) or column (vertical)] of Items.

  • By understanding how to nest Items and Containers, you can produce ANY arrangement of Elements (usually divs) any way you like.
    Note: Scrolling inside divs is accomplished through standard methods.

  • A Layout (Object/Class) has only two properties:
    1. A root Item - representing the "arrangment" of Elements created above.
    2. An anonymous function with Elements current width and height as arguments, to determine if this "arrangment" of Elements. Should be used. For Example:
      (x, y) => { ((x < 1000) && (x < 1000)) ? true : false }
      ...would use this layout if (most likely - you choose) screen size width < 1000. and height < 1000

  • A Handler (Object/Class) represents a choice list of which Layout to display.(Array <Layout>)
    After each resizeEvent, this Handler instance iterates over a list of Layouts, checking the Layout.conditionalFunction. if it returns true, that Layout.container is Chosen.
    If it makes it to the end of the list, and no layout is chosen, it defaults to the last Layout in list, then whines about it in console.log()
    If a liefs-layout-manager Item, is the root of the page, there will be only one Handler instance.
    If a liefs-layout-manager Item is a child of *some other resize method* then a seperate Handler instance must be made for each "free Item child" of the page.
